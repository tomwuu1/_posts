# 发布前要求

1. case全通过，bug已修复
2. 监控梳理
3. 梳理上下游系统依赖
4. 确认发布步骤（代码，db，配置等）
5. 确认回滚步骤（代码，db，配置等）

# 发布中

1. 发布期间禁止其他事情干扰
2. QA按照发布顺序发布（优先灰度观察单机日志，监控）5min-10min
3. qt，wx群周知发布应用
4. QA&Dev查看监控-查看30min以上
5. QA&Dev查看日志-查看30min以上
6. 发现问题回滚（蓝绿发布），或开关降级
